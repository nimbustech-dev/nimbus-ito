# Deal-ito 통합 실적/이익 대시보드

엑셀 실적 데이터를 기반으로 **매출·이익**을 분석하고, 담당자별 목표 대비 달성률·월별 현황·순위 조회를 제공하는 Streamlit 웹 애플리케이션입니다.

## ⚖️ 실적 반영 로직

담당자 역할에 따라 **동일 비율**로 매출·이익 실적을 배분합니다.

| 역할 | 컬럼 | 반영 비율 |
|------|------|----------|
| **고객** | `Deal - 담당자_고객` | **40%** |
| **관리** | `Deal - 담당자_관리` | **30%** |
| **소싱** | `Deal - 담당자_소싱` | **30%** |

예: 매출 1,000만원 Deal의 경우  
→ 고객 400만원, 관리 300만원, 소싱 300만원으로 반영

## 📊 월별 매출/이익 계산법

**월별 매출/이익 조회** 메뉴에서 사용하는 계산 방식입니다. 필수 컬럼 유무에 따라 두 가지 방식 중 하나가 적용됩니다.

---

### 방식 1: 기본 공식 (권장)

**적용 조건**: `Deal - @MM (연도별)`, `Deal - @매출액 (연도별)`, `Deal - @이익 (연도별)`, `Deal - 실투입 (01)~(12)` 컬럼이 모두 있을 때

| 항목 | 공식 |
|------|------|
| **m월 매출** | Σ ( `실투입 (0m)` ÷ `@MM (연도별)` × `@매출액 (연도별)` ) |
| **m월 이익** | Σ ( `실투입 (0m)` ÷ `@MM (연도별)` × `@이익 (연도별)` ) |
| **m월 MM** | Σ `실투입 (0m)` |

> `(0m)` = `(01)`, `(02)`, … `(12)` (월 번호 두 자리)

**예시**: Deal A — MM 3.94, 연간 매출 26,004,000원, 3월 실투입 0.94  
→ 3월 매출 = 0.94 ÷ 3.94 × 26,004,000 ≈ 6,204,000원

---

### 방식 2: 폴백 (월별 컬럼 직접 사용)

**적용 조건**: 위 기본 컬럼이 없을 때

| 항목 | 공식 |
|------|------|
| **m월 매출** | Σ `Deal - @월별매출 (01)~(12)` |
| **m월 이익** | Σ `Deal - @월별이익 (01)~(12)` |
| **m월 MM** | Σ `Deal - 실투입 (01)~(12)` (있으면) 또는 연간 MM을 매출 비율로 월별 배분 |

---

### 보조 계산

- **월별 이익률** = (m월 이익 ÷ m월 매출) × 100
- **인당 매출/이익**: 전체 MM > 0일 때만 `전체 매출÷전체 MM`, `전체 이익÷전체 MM`로 산출

## 📈 전체 분석 계산법

전체 실적 대시보드의 **전체 분석** 탭에 표시되는 MM·매출·이익·이익률의 계산 과정을 설명합니다.

### 사전 단계

1. **대상 Deal 필터링**
   - 선택한 담당자(고객/관리/소싱)가 **한 명이라도 포함된 Deal**만 분석 대상
   - `Deal - 담당자_고객`, `Deal - 담당자_관리`, `Deal - 담당자_소싱` 중 하나라도 선택 담당자와 일치하면 포함

2. **선택기간 매출·이익 산출**
   - **1~12월 전체 조회 시**: `Deal - @매출액 (연도별)`, `Deal - @이익 (연도별)` 값 그대로 사용
   - **특정 월만 조회 시**: `Deal - @월별매출 (01)~(12)`, `Deal - @월별이익 (01)~(12)` 중 해당 월 컬럼을 합산
   - 금액 컬럼은 `clean_currency_val`로 숫자만 추출한 후 사용

3. **Deal 기준 집계**
   - 동일 Deal이 여러 행(People별)으로 나뉘어 있어도 Deal 단위로 하나로 묶음
   - `Deal - RecordId` 또는 `Deal - 이름`으로 그룹핑, 각 컬럼은 `first` 값 사용

---

### 1. MM 분석

| 지표 | 계산 공식 |
|------|----------|
| **MM 전체값** | Σ (각 Deal의 `Deal - @MM (연도별)` 값) |
| **인당 MM평균값** | MM 전체값 ÷ Deal 개수 (Deal당 평균 MM) |

- `Deal - @MM (연도별)` 컬럼 값을 `clean_currency_val`로 숫자만 추출 후 합산·평균 계산

---

### 2. 매출 분석

| 지표 | 계산 공식 |
|------|----------|
| **매출 전체값** | Σ (각 Deal의 `선택기간_총매출`) |
| **인당 매출 평균값** | 매출 전체값 ÷ Deal 개수 (Deal당 평균 매출) |

- `선택기간_총매출`은 위 "선택기간 매출·이익 산출" 단계에서 구한 Deal별 매출 합계

---

### 3. 이익 분석

| 지표 | 계산 공식 |
|------|----------|
| **이익 전체값** | Σ (각 Deal의 `선택기간_총이익`) |
| **인당 이익 평균값** | 이익 전체값 ÷ Deal 개수 (Deal당 평균 이익) |

- `선택기간_총이익`은 위 "선택기간 매출·이익 산출" 단계에서 구한 Deal별 이익 합계

---

### 4. 이익률

| 지표 | 계산 공식 |
|------|----------|
| **이익률** | (이익 전체값 ÷ 매출 전체값) × 100 |

- 매출 전체값이 0이면 이익률은 0%

---

### 계산 흐름 요약

```
조회 조건(연도, 기간, 담당자) 적용
    ↓
대상 Deal 필터링 (담당자 포함 여부)
    ↓
Deal별 선택기간_총매출·선택기간_총이익 산출
    ↓
Deal 단위 집계 (agg_df)
    ↓
MM 전체값 = Σ MM
MM 평균값 = mean(MM)
매출 전체값 = Σ 선택기간_총매출
매출 평균값 = mean(선택기간_총매출)
이익 전체값 = Σ 선택기간_총이익
이익 평균값 = mean(선택기간_총이익)
이익률 = 이익 전체값 / 매출 전체값 × 100
```

## ❓ 자주 묻는 질문

### Q. 로컬과 배포 사이트 값이 다릅니다

코드 수정 후 서버에서 **Streamlit 프로세스를 재시작**했는지 확인하세요. `git pull`만으로는 반영되지 않습니다.

### Q. 엑셀 업로드 후 데이터가 안 나옵니다

필수 컬럼(`Deal - 이름`, `Deal - 담당자_*`, `Deal - @매출액 (연도별)` 등)이 엑셀에 있는지 확인하세요. 컬럼명은 대소문자·띄어쓰기까지 일치해야 합니다.

### Q. 월별 매출/이익이 0으로 나옵니다

다음 컬럼을 확인하세요.

- **기본 공식**: `Deal - @MM (연도별)`, `Deal - @매출액 (연도별)`, `Deal - @이익 (연도별)`, `Deal - 실투입 (01)~(12)`
- **폴백**: `Deal - @월별매출 (01)~(12)`, `Deal - @월별이익 (01)~(12)`

### Q. `dashboard_cache.pkl` 충돌로 pull이 안 됩니다

```bash
git checkout -- dashboard_cache.pkl
git pull origin main
```
